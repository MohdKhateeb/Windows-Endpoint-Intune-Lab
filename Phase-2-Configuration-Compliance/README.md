# Phase 2 – Configuration & Compliance
 
This phase focuses on configuring Windows devices using Microsoft Intune and enforcing compliance policies.
 
After devices are enrolled, administrators must configure device behavior, enforce security policies, and control access to corporate resources.
 
---
 
## Labs Included
 
LAB-WIN-006 – Device Configuration Profiles  
LAB-WIN-007 – Compliance Policies  
LAB-WIN-008 – Conditional Access with Device Compliance  
LAB-WIN-009 – Advanced Configuration using Settings Catalog
 
---
 
## Learning Objectives
 
After completing this phase you will understand:
 
- How configuration profiles manage device settings
- The difference between **Configuration Policies and Compliance Policies**
- How Conditional Access enforces Zero Trust security
- How Settings Catalog enables granular device configuration
 
---
 
## Enterprise Relevance
 
These configurations are widely used in enterprise environments for:
 
- Device security enforcement
- Policy standardization
- Conditional access security controls
- Compliance-based access enforcement
 
This phase represents the **core operational management layer of Microsoft Intune**.
